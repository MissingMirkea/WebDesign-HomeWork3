game mode: weight calculator

Every CSS selector has a weight: inline > ID > class/pseudo-class > element. See how the cascade resolves conflicts — then test yourself below with the Paragraph D challenge!

📐 Specificity scoring (from low to high)
Selector type Example Weight (a,b,c)
Element / pseudo-element p, ::before (0,0,1)
Class / attribute / pseudo-class .info, [type], :hover (0,1,0)
ID selector #warning (1,0,0)
Combined (ID + class) #warning.info (1,1,0) → higher than ID alone

🧠 how specificity works

When multiple rules target same element, browser computes selector weights: ID beats class, class beats element.
If equal? last declaration wins.

#warning.info → (1,1,0)
#warning → (1,0,0)
.info → (0,1,0)
p → (0,0,1)

✨ In our game: paragraph D uses all three selectors — the most specific rule #warning.info overrides ID and class declarations.

✅ Paragraph B receives properties from element + class (.info): color: steelblue, font-size: 1.25rem, background-color: #e8f4fd, padding: 8px 14px, border-left: 5px solid steelblue, font-family: Arial, sans-serif (from .info), and line-height: 1.4 (from p).

✅ Paragraph D receives highest combined selector (#warning.info): color: darkorange, background: #fff7e0, border: 2px solid darkorange, font-size: 1.1rem, font-style normal, letter-spacing: 0.03em. Font-family from #warning (Courier New) and line-height from p.

🎯 Specificity wins: element (0,0,1) < class (0,1,0) < ID (1,0,0) < ID+class (1,1,0). Paragraph D rule overrides both ID and class declarations.
